HTML


  • HTML stands for Hypertext Markup Language and it gives the content structure.
  • The head element contains information about the webpage.
  • The Element contains information, or metadata, that helps the browser to render the page correctly. Elements included within the are not visible when a user looks at a webpage.
  • The body element represents the visible content shown to the user.
  • The element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • HTML attributes are always defined in the beginning element, never the closing element.

CSS


  • To add style to our webpage, we will use Cascading Style Sheets (CSS), which is a programming language used by developers to define how webpage content is presented to users.
  • Whereas HTML strictly handles the structure and content of a webpage, CSS defines the colors, aesthetics, visual layout, and organization.
    • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet. Let's explore these and then decide which approach we will use.

Git


  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git is a version control system that lets you manage and keep track of each version of your project. To access Git, we use the command line.
  • However, Git commands are not the same as the bash commands such as cd and mkdir that we use to navigate and create directories or perform other tasks.
  • Git commands always start with git and are used to save and track versions of your code.
  • Using Git commands, we can initiate (or create) a repo, add our code to the repo, and view the different versions of our code, which are each stored in a unique commit.
